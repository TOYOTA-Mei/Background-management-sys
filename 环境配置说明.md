# 前后分离系统 - 环境配置说明

## 快速开始

### 1. 环境要求
- Python 3.8+
- MySQL 5.7+ (或使用SQLite)
- Git

### 2. 安装依赖
```bash
pip install -r requirements.txt
```

### 3. 配置环境变量

**方式一：使用.env文件（推荐）**
```bash
# 复制环境配置模板
copy .env.example .env

# 编辑.env文件，填写实际配置
# Windows: notepad .env
# macOS/Linux: nano .env
```

**方式二：直接设置环境变量**
```bash
# Windows PowerShell
$env:DB_PASSWORD="your_password"
$env:DEBUG="True"

# Linux/macOS
export DB_PASSWORD="your_password"
export DEBUG="True"
```

### 4. 配置项说明

#### 核心配置
```env
# Django密钥（生产环境务必修改）
SECRET_KEY=your-secret-key

# 调试模式（生产环境设为False）
DEBUG=True

# 允许的主机（用逗号分隔）
ALLOWED_HOSTS=localhost,127.0.0.1,你的域名
```

#### 数据库配置
```env
# MySQL配置
DB_ENGINE=django.db.backends.mysql
DB_NAME=db_admin
DB_USER=root
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=3306

# SQLite配置（如果使用）
# DB_ENGINE=django.db.backends.sqlite3
# DB_NAME=db.sqlite3
```

#### 缓存配置
```env
# 缓存过期时间（秒）
CACHE_TIMEOUT=3600
```

### 5. 创建数据库
```sql
-- MySQL命令行
mysql -u root -p
CREATE DATABASE db_admin CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 6. 运行迁移
```bash
python manage.py makemigrations
python manage.py migrate
```

### 7. 创建超级管理员
```bash
python manage.py createsuperuser
```

### 8. 启动服务器
```bash
python manage.py runserver
```

访问 http://localhost:8000/swagger/ 查看API文档

## 安全建议

1. **生产环境必做**
   - 修改 `SECRET_KEY`
   - 设置 `DEBUG=False`
   - 配置正确的 `ALLOWED_HOSTS`
   - 使用强密码

2. **数据库安全**
   - 不要使用root用户，创建专用数据库用户
   - 设置强密码
   - 定期备份数据

3. **环境变量**
   - `.env`文件不要提交到Git
   - 生产环境使用环境变量或密钥管理服务

## 常见问题

### Q: 提示找不到.env文件
A: 确保`.env`文件在项目根目录，且文件名正确

### Q: 数据库连接失败
A: 检查MySQL服务是否启动，配置是否正确

### Q: 依赖安装失败
A: 确保Python版本>=3.8，尝试升级pip: `pip install --upgrade pip`
